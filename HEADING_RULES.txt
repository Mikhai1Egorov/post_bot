# 🔠 HEADING_RULES — Structured Specification

## 1. Purpose

Defines how headings and subheadings are structured within an article.

The goal is to ensure:

* readability
* logical content segmentation
* clear visual hierarchy

These rules prevent unstructured “wall of text” outputs and improve user experience. 

---

## 2. What Is Regulated

The system enforces:

* presence of a main title
* hierarchical structure of headings:

  * `h1` — main title
  * `h2` — section headers
  * `h3` — subsection headers
* logical segmentation of content
* elimination of large unbroken text blocks

---

## 3. Key Principles

* Every article MUST have a clear heading structure
* Content MUST be divided into logical sections
* Headings MUST reflect semantic meaning, not just formatting
* Structure MUST improve readability, not complicate it

---

## 4. Heading Hierarchy Model

### Standard Hierarchy

```html
<h1>Main Title</h1>

<h2>Section</h2>
<p>...</p>

<h3>Subsection</h3>
<p>...</p>
```

### Rules

* Only one `h1` per article
* `h2` defines main sections
* `h3` refines sections when needed
* Do NOT skip levels (e.g., `h1 → h3`)

---

## 5. How It Works in the System

### Stage 1 — Prompt Level

* Prompt-template instructs LLM to:

  * include headings
  * structure content into sections

### Stage 2 — Post-Processing

The system:

* validates heading presence
* checks hierarchy correctness
* fixes structure if needed
* normalizes HTML tags

---

## 6. Where It Is Used

* `prompt-template` (generation stage)
* `post-processing` module (validation + correction)

---

## 7. System Behavior

The system automatically:

* detects missing headings
* inserts or restructures headings if necessary
* splits large text blocks into sections
* ensures consistent hierarchy

---

## 8. Relationship with Other Modules

Heading rules are tightly connected with:

* `ARTICLE_STRUCTURE_TEMPLATES.txt` — defines logical sections
* `CONTENT_LENGTH_RULES.txt` — influences section size
* `TONE_OF_VOICE_RULES.txt` — affects heading phrasing
* `HTML_FORMATTING_RULES.txt` — ensures valid HTML output
* `SYSTEM_INSTRUCTIONS.txt` — enforces consistency

---

## 9. Output Requirements

Final article MUST:

* contain a clear title (`h1`)
* include structured sections (`h2`, optional `h3`)
* avoid long unbroken paragraphs
* be easy to scan visually

---

## 10. Key Insight (Core Rule)

```html
Good headings are not decoration — they are structural anchors of the article.
```

---

## 11. One-Line Summary

The system enforces a clear hierarchical heading structure to ensure readability, logical flow, and properly formatted HTML output.

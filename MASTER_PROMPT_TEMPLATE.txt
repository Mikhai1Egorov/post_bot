# 🧠 MASTER_PROMPT_TEMPLATE — Structured Specification

## 1. Purpose

Defines the core content generation prompt used by the system.

This is the **base template** that:

* injects task data into generation
* enforces universal writing constraints
* ensures human-like, high-quality output

---

## 2. Role in System

This template is part of:

```text id="flow1"
FINAL PROMPT =
SYSTEM_INSTRUCTIONS
+ STYLE_TEMPLATE
+ MASTER_PROMPT_TEMPLATE
+ TASK_DATA
+ CONTENT_LENGTH_RULES
+ OPTIONAL_BLOCKS
```

It acts as the **central content instruction layer**.

---

## 3. Input Variables

The template consumes task data:

* `{topic}`
* `{title}`
* `{keywords}`
* `{response_language}`
* `{search_language}`
* `{time_range}`

---

## 4. Title Logic

* If `{title}` is provided → use it
* If empty → generate a relevant title based on topic

---

## 5. Language Enforcement

```text id="lang1"
Output MUST be strictly in {response_language}
```

* no mixing languages
* no fallback behavior

---

## 6. Content Requirements

The generated text must be:

* human-like
* logical
* natural
* non-repetitive
* concise (no fluff)

---

## 7. Keywords Usage

Rules:

* integrate `{keywords}` naturally
* avoid keyword stuffing
* maintain readability

---

## 8. Source Awareness

If research context exists:

* use it as factual base
* respect time constraint:

```text id="time1"
{time_range}
```

* prioritize relevant and recent information

---

## 9. Prohibited Behavior

The model MUST NOT:

* mention AI or generation
* explain the writing process
* use meta phrases like:

  * “in this article”
* produce шаблонный (template-like) text

---

## 10. Structural Requirements

The output MUST include:

* title
* subheadings
* paragraphs
* lists (if appropriate)

Structure must be:

* logical
* readable
* well segmented

---

## 11. Formatting Rules

The text must:

* be clean
* contain no extra symbols
* be ready for HTML transformation
* align with:

  * heading rules
  * structure templates

---

## 12. Output Constraint

```text id="out1"
Return ONLY final article text.
No explanations.
No metadata.
```

---

## 13. System Behavior

The template ensures:

* consistent generation quality
* alignment with task parameters
* compatibility with post-processing
* readiness for HTML rendering

---

## 14. Relationship with Other Modules

This template works together with:

* `SYSTEM_INSTRUCTIONS.txt` — global constraints
* `PROMPT_TEMPLATE_REGISTRY.txt` — style resolution
* `JOURNALIST_PROMPT_STYLE.txt` — style profile (`journalistic`)
* `SIMPLE_PROMPT_STYLE.txt` — style profile (`simple`)
* `EXPERT_PROMPT_STYLE.txt` — style profile (`expert`)
* `CONTENT_LENGTH_RULES.txt` — content size
* `LENGTH-BLOCKS.txt` — optional blocks logic
* `ARTICLE_STRUCTURE_TEMPLATES.txt` — layout
* `HTML_FORMATTING_RULES.txt` — final rendering

---

## 15. Key Insight

```text id="ins1"
This template defines WHAT to write.
Style templates define HOW to write.
```

---

## 16. One-Line Summary

The master prompt template injects task data and enforces core writing rules to produce clean, human-like, structured article content.

# 🧾 HTML_TEMPLATE — Structured Specification ## 1. Purpose Defines the canonical HTML document template used to render generated articles. This template ensures: * consistent layout * structured metadata * compatibility with publishing systems * predictable rendering across platforms --- ## 2. Template Type * Full HTML document (``) * Used in **Post-Processing stage** * Final artifact for: * publishing * archive (ZIP) * preview --- ## 3. Dynamic Variables The template uses placeholders injected by the system. ### Core Variables * `{{title}}` * `{{meta_description}}` * `{{keywords}}` * `{{channel}}` * `{{style}}` * `{{length}}` * `{{mode}}` * `{{time_range}}` * `{{search_language}}` * `{{response_language}}` --- ## 4. Document Structure ### 4.1 Root Structure ```html id="q3n8dr" ... ... ``` --- ### 4.2 Head Section Contains: * charset * viewport * SEO metadata * system metadata (`x-*` fields) * inline CSS ### Metadata Purpose ```html id="4kl3u2" ``` Used for: * debugging * tracking * downstream integrations --- ### 4.3 Body Structure ```html id="twl8m1"

{{title}}

``` --- ## 5. Logical Blocks --- ### 5.1 Header Block Includes: * title (`h1`) * metadata (channel, schedule) * lead paragraph --- ### 5.2 Image Block (Optional) Condition: ```text id="img1" include_image = true ``` The condition is evaluated on the normalized internal flag derived from Excel `include_image` (`TRUE`/`FALSE`). Rendered as: ```html id="img2"
``` If false: * block is NOT rendered --- ### 5.3 Content Block ```html id="cnt1"

``` Contains: * processed HTML from LLM output * tags: * `h2`, `h3` * `p` * `ul`, `li` --- ### 5.4 Footer Block (Optional) Condition: ```text id="ftr1" footer_text OR footer_link provided ``` Variants: **Text only:** ```html id="ftr2"

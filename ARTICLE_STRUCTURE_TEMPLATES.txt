# 📄 ARTICLE_STRUCTURE_TEMPLATES — Structured Specification

## 1. Purpose

Defines how an article is structured — i.e., which logical blocks it consists of.

The structure ensures consistency, readability, and alignment with the selected style and task type. 

---

## 2. Inputs

The structure is determined dynamically based on:

* `style` (from Excel)
* task type (implicit from input context)

---

## 3. Conceptual Structure (Abstract Example)

```
Title
Introduction
Main Content
Conclusion
```

This is a **reference structure**, not a strict template. 

---

## 4. Key Principles

* Structure is **NOT globally fixed**
* Structure is **resolved dynamically**
* Structure depends on:

  * selected `style`
  * prompt-template logic
  * task context
* Different tasks may produce **different article structures**

---

## 5. How It Works in the System

### Resolution Flow

```
task → style → prompt-template → structure → generation
```

* The system does NOT hardcode structure
* Structure is embedded inside the **prompt-template**
* The LLM follows the structure defined by the prompt

---

## 6. Where It Is Used

* Inside `prompt-template`
* During prompt construction (Prompt Resolver stage)

---

## 7. Relationship with Other Modules

Article structure is tightly connected with:

* `TONE_OF_VOICE_RULES.txt` (how text sounds)
* `CONTENT_LENGTH_RULES.txt` (how much content is generated) 
* `HEADING_RULES.txt` (how headings are structured) 
* `HTML_FORMATTING_RULES.txt` (final rendering)
* `SYSTEM_INSTRUCTIONS.txt` (global constraints)

---

## 8. System Behavior

The system:

* selects structure via prompt-template
* enforces presence of:

  * title
  * logical sections
  * readable flow
* allows flexible composition depending on task

---

## 9. Output

The LLM generates:

* a fully structured article
* with logical sections
* aligned with the selected style
* ready for HTML post-processing

---

## 10. Key Insight (Core Rule)

```
Structure is NOT a fixed template.
Structure is a function of style + prompt-template + task context.
```

---

## 11. Summary (One Line)

The system dynamically builds article structure through prompt templates, ensuring flexibility, consistency, and alignment with task parameters.
